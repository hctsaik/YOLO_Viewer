# 需求:縮圖牆標記兩張影像做疊圖比較(取代雙 model 資料夾比較)

## User 原話
> 「比較模式的功能,我想要把原本的資料夾疊圖功能改成是,我在縮圖前裡面標註兩張影像來做疊圖就好
> (不再需要兩個資料夾)。」

## 背景
目前「🔀 比較模式」是**兩個 model 對同一包影像**的覆蓋率/分歧比對(需要 sidebar 填「第二個 model
結果資料夾 B」)。User 認為這個「需要兩個資料夾」的設計太重,想改成:**只用一個資料夾**,直接在
縮圖牆裡挑兩張影像做疊圖比較。

## AskUserQuestion 確認(2026-07-05)
1. **疊圖要顯示什麼**:兩者都要,可切換 ——
   - 像素疊合(並排/差異/混合),比較兩張影像本身的像素內容。
   - 偵測框疊合(A 藍/B 橘),兩張影像各自的 YOLO 偵測框疊在一起比對。
2. **現有雙 model 覆蓋率儀表板 + 分歧佇列(modeldiff)怎麼處理**:整個移除,改成新的兩圖疊圖模式。

## 範圍(PO 裁決,見 ROADMAP 決策日誌)
- 移除:sidebar「第二個 model 結果資料夾 B」欄位、覆蓋率儀表板、分歧 triage 佇列、`modeldiff` 在
  app.py 的整合(`modeldiff.py`/`test_modeldiff.py` 本體保留不刪,比照 framecompare/framediff 的
  既有慣例:程式碼保留、未來可回退,只是 app.py 不再呼叫)。
- 新增:縮圖牆(主縱向牆)每張縮圖左下角一個「標記」小圖示,最多同時標記 2 張(①藍/②橘);
  標記滿 2 張後,「🔀 比較模式」toggle 顯示疊圖比較視圖,可切換「像素疊合」/「偵測框疊合」。
- 不需要第二個資料夾;兩張標記影像都來自同一個(唯一的)影像資料夾。
