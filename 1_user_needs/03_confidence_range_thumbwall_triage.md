# 需求:信心門檻要卡縮圖牆、且要雙邊

> `/user` 階段產物:我只講「我想要什麼、為什麼」。怎麼實作交給 PO/architect。

## 1. 痛點

現在頂列的「信心門檻」slider 只有一個值(下界),而且它只影響**主圖裡疊的框**——
左邊縮圖牆永遠列出資料夾裡的**每一張圖**,不管這張圖有沒有落在我設的信心範圍內。

我要用信心門檻來做的事情是「篩掉不相關的圖」,不是「篩掉一張圖裡的某幾個框」。
現在的行為讓我還是要一張一張切過去,才知道哪張其實沒有我在意的偵測。

## 2. 我想要的

1. **信心門檻要能設上下兩個界線**(下界–上界),不是只有一個值。
2. **信心門檻要卡縮圖牆**:縮圖牆(以及切圖的導覽清單)只列出「至少有一個偵測的信心落在我設的區間內」的圖。
   不符合的圖**直接從清單消失**,不是還在清單裡但變灰或框不顯示。
3. **完全沒有偵測框的圖,一併濾掉**——這種圖不可能有偵測落在任何信心區間內,邏輯上本來就該濾掉,
   不需要另外開關。

## 3. 我眼中的「成功」

- 縮圖牆張數會隨我調整的區間變多變少;把區間縮到很窄,牆上就只剩下少數幾張。
- 我把區間拉回 0.00–1.00(全開),牆上恢復列出所有「至少有框」的圖(0 框的圖仍不出現——本來就沒東西可篩)。
- 主圖裡的疊框行為維持不變(仍然只畫落在我設區間內的框),這點跟現在一樣,我沒有意見要改。

## 4. 不在乎 / 可放棄

- 我不要求縮圖牆裡「有框但框不在區間內」的部分被特別標示——那張圖只要有**任何**一個框落在區間內就該出現,
  出現之後縮圖上照舊燒全部框給我看(這是現有行為,我沒說要改)。
- 比較模式(雙 model 覆蓋比對)那邊已經有雙邊信心 slider,我這次講的是**單張檢視模式**的頂列信心門檻,
  沒有要動比較模式。
