# 需求:版面重設計第 5 輪 — image-viewer-first + 可收疊

> `/user` 階段產物:我只講「我想要什麼、為什麼、怎樣算成功」。怎麼實作交給 PO/architect。
> 但我把每個訴求對齊到「現在 app 哪裡讓我痛」,讓下游有可驗收的判準。
> 這是第 5 輪;前 4 輪候選都被我退。退的原因附在每節,免得下游又踩同一顆釘子。

## 1. 我是誰 / 在什麼處境用

我是缺陷審圖員。一天裡我 90% 的時間在做同一件事:**看這張圖 → 看框對不對 → 切下一張**。
我的手大部分時間在滑鼠(縮放、平移、hover 讀值)和方向鍵(切圖)上。
我偶爾(一天幾次、特定時點)才會去比較前後張、查相似、看漏檢佇列、改篩選、匯出。

**核心訴求(這輪唯一判準):它大部分時間就是個 image viewer。** 看圖要最大、最乾淨、零摩擦;
少用功能該收起來、需要時一鍵叫出來,平時不要佔我的畫面、不要分散我的注意力。

## 2. 我想達成的(Jobs To Be Done)

- 我想要**畫面打開就是一張大圖**,viewer 佔滿我的螢幕,不被一堆 expander、slider、欄位擠到變小。
- 我想要**用方向鍵連續切圖**(←/→),連切 20 張不卡、不斷、不用每張都重新點一下畫面找焦點。
- 我想要**隨時看到偵測框、隨手調信心門檻**——這是我每天的核心動作,不是設定一次的偏好。
- 我想要**少用功能收起來**:比較、相似、聚類、漏檢佇列、匯出、篩選、資料源、標記判定——
  平時不出現,我要用的時候一鍵展開,用完關掉,畫面回到乾淨的看圖狀態。
- 我想要**每張必做的判定(對/錯/反光)永遠在同一個固定、看得到的位置**,手不用找。

## 3. 痛點(對著現在的 app 講,不是空想)

- 現在畫面太雜:頂部一排控制、全寬漏檢佇列、三欄、底下又掛比較/相似/DZI/聚類/匯出一堆 expander。
  圖被夾在中間,很多時候我只想看圖,卻要先用眼睛跳過一堆我這一刻用不到的東西。
- 現在切一張圖整個 viewer 會「閃一下、重來」,我連按方向鍵切圖會走兩步斷一次,比按鈕還累。
- 判定鈕在右欄中段,新圖每張我都要把視線移過去找它,動線不順。

## 4. 我提的版面(第 5 輪)— 預設顯示什麼、什麼收疊、用什麼收、為什麼

> 視角已對齊 Streamlit 1.56 真能做到的:sidebar 可收、expander、tabs、popover、dialog、
> segmented_control、toggle、session_state、可隱藏 sidebar、自訂元件(iframe)給最大寬高。
> 我也看過現在的 app.py / viewer 元件 / thumbwall 元件,所以下面每條都標「為什麼這在現有結構下成立」。

### 4.1 預設「日用層」(永遠在、零摩擦)——只有這些常駐

1. **主 viewer 吃掉最大畫面**:預設**收起 Streamlit sidebar**(把資料源/篩選/排序都塞進去),
   主區用「縮圖窄欄 + viewer 寬欄 + 判定窄欄」,viewer 那欄要拿到最大寬高(iframe 高度給大,
   讓我一眼就是大圖)。少用面板**不渲染**在主流程裡,所以畫面真的空出來。
2. **頂部一條極窄常駐列**,只放我每秒在用的東西,從左到右:
   - 上一張 / 下一張按鈕;
   - **進度直接可點即就地編輯的跳頁**(顯示像 `127 / 850`,點它就地變成輸入框打數字 Enter 跳過去)。
     → 退掉第 4 輪:不要把跳頁藏進 popover 要先按 j 再輸入再 Enter(三步)。我看到漏檢佇列說「第 37 張」
     時要能**一步**跳過去,所以跳頁必須常駐、不能藏。
   - **偵測框 toggle(開/關)** + **信心門檻**的**常駐極薄載體**(slider 或 −/+ 微調)。
     → 退掉第 3 輪:門檻對我是**高頻微調**(不同 lot 雜訊不同),不是設一次。toggle 可以收,
     **slider/門檻不准收進 popover**;每次調都先點開面板=不可接受。留一條常駐薄 slider 或熱鍵 −/+。
3. **左:縮圖牆**(整張可點、selected 高亮、疊框、⭐/✓ 標記、偵測數徽章)——保留,這是我找圖的眼睛。
4. **右:判定列**——**verdict(對/錯/反光)放在這一列最上方、首屏一定看得到、不准折行**。
   → 退掉第 6 輪:窄欄把 verdict 折到底部 strip 不行。verdict 是我每張必填的第一動作,
   位置要固定、要在首屏、1080p 與 1440p 都不折行。bookmark / status / tags / comment / 儲存 接在它下面。

### 4.2 「召喚層」(平時不在,一鍵展開,用完收回)——這些全部收疊

我可以接受用 **popover / dialog / 一個統一的工具區 tabs** 把這些收起來,平時零畫面成本:
- 前後比較 framecompare、找相似 simhash、聚類 embcluster、漏檢佇列 missedq、匯出、DZI 瓦片。
- 篩選 / 搜尋 / 排序 / 資料源設定 / overlay 類別篩 → 收進 sidebar(預設收合)。

**要求:召喚層沒被打開時,它的重內容不要被算**(不要每次切圖都偷算比較圖、相似、聚類、匯出資料)。
平時我只切圖看圖,那些就該是 0 成本;我點開哪個才算哪個。

### 4.3 鍵盤連切——這是這輪的命根,前 4 輪都沒解決好

我最高頻的動作是**方向鍵連切圖**。我要求做到:
- 我**連按 ←/→ 切 20 張**,每一次都生效、不漏、不需要中間用滑鼠點一下畫面「找回焦點」。
- 切圖時 viewer**不要整個重來/閃一下**;它應該是「同一個 viewer 換了張圖」,不是「砍掉重建一個 viewer」。
  → 我看到現在每切一張、viewer 那塊的識別就變了,導致它被銷毀重建、焦點掉、取樣重算。
    這是物理前提就不對,不是「關面板後再 focus 一下」能補的。**這顆釘子必須在這輪拔掉**:
    (a) 讓 viewer 跨切圖**存活**、不要每張重建;
    (b) 每次重畫後,viewer 自己**主動把鍵盤焦點抓回畫面**,讓我下一次按鍵一定有作用;
    (c) 要**證明**連切時第一次按鍵不會落空(不是嘴上說,要量得出來)。
- 鍵盤改判定(像 1/2/3 設 verdict、e 切下一張)我可以要,但**必須安全**:
  → 我在 viewer 裡用滑鼠拖曳平移看細節時,手指還壓著想微調,**不小心碰到 1/2/3 不能默默改掉判定、
    碰到 e 不能默默跳走**。任何鍵盤改的判定/跳轉都要**給我看得到的回饋**(像一閃的提示),
    而且**鍵盤改的判定要能被撤銷**(進 undo 軌跡)。

### 4.4 撤銷(undo)——語義要對審圖

我連判十張,發現第 6 張判錯要回去改。我要的 undo 是:
- **撤銷 = 回到那一張圖、並還原那張的值**,不是只默默改個值卻不把我帶回去。
  否則我撤了也不知道撤的是哪張、改了什麼。撤銷要能跨圖、且要把我跳轉回該圖。

## 5. 我眼中的「成功」(可觀察、可被下游當驗收)

1. 打開 app,**首屏就是大圖**;我這一刻用不到的功能(比較/相似/聚類/漏檢/匯出/篩選)**畫面上看不到**,
   要用時一鍵叫得出來、用完收得回去。
2. 我**連按 ←/→ 切 20 張**,每一張都換成功、**中途不用碰滑鼠找焦點**,viewer 不閃爍重建。
   （下游要**量得出來**「連切不漏、不重建」,不能只用嘴保證;前 4 輪栽在這——這輪要先講清楚怎麼量。）
3. **偵測框門檻**我隨手就能調(常駐薄 slider 或 −/+ 熱鍵),**不用先點開任何面板**;toggle 收掉沒關係。
4. 看到「第 N 張」時,我**一步**就跳過去(進度可點即就地輸入),不用三步。
5. **verdict(對/錯/反光)固定在同一個首屏看得到的位置、不折行**,每張新圖我手不用找它;
   在 1080p 和 1440p 都成立。
6. 鍵盤改判定 / 跳轉**有看得見的回饋**,而且**可撤銷**;撤銷會**把我帶回那張圖並還原值**。
7. 召喚層沒打開時**不偷算**——我平時只切圖看圖,切圖該是順的,不被沒打開的比較/聚類/匯出拖慢。

## 6. 我不在乎 / 可放棄的(給 PO 砍的空間)

- 我**不在乎**召喚層長得多漂亮、是 popover 還 dialog 還統一工具 tabs——隨你選最好實作的,
  只要「平時不佔畫面、一鍵叫得出、用完收得回、沒開不偷算」。
- 我**不要求**召喚層裡的功能彼此同框並排;一次用一個就好。
- 我**可以接受**少用設定(資料源/篩選/排序)藏進收合的 sidebar,要用再拉開。
- 我**不在意**動畫過場、主題配色、好不好看的細節——先把「看圖零摩擦 + 連切不斷 + 門檻隨手調 +
  判定固定可見 + 撤銷對」這五件做對,其它都次要。

## 7. 給下游的提醒(免得這輪又被退)

前 4 輪我退的點,這輪請在交給 architect 之前就有交代,否則我還是會退:
- **鍵盤連切**:viewer 跨切圖存活(不每張重建)+ 重畫後主動抓焦點 + 證明首次按鍵不落空。三點缺一不可。
- **焦點守衛**:不能只擋輸入框;我在畫布上拖曳平移時誤觸 1/2/3/e 不能默默改判定/跳走——要回饋 + 可撤銷。
- **門檻不准收**:toggle 可收,信心門檻要留常駐薄載體或 −/+ 熱鍵。
- **跳頁不准退化**:常駐、一步可達(可點即就地輸入),別藏進 popover。
- **undo 跨圖語義**:撤銷=回到那張圖並還原值,要明講。
- **verdict 位置**:首屏固定可見、不折行,1080p/1440p 都要。
- **可量測**:「連切不漏」「沒開不偷算」這種硬判準,要在這輪就指定**怎麼客觀量**(不是放寬斷言含混過去),
  不然就是空頭支票。現在的 E2E 是冷啟、沒有計時/連切計數的工具,這缺口要補上才收得了尾。
