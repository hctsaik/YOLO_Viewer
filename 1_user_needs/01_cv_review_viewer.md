# 需求:CV Expert Review Viewer(暫名 CV Review Workbench / Vision Case Explorer)

> 本檔是 `/user` 階段產物:只講「想要什麼、為什麼痛」,不碰技術實作。技術決策交給 `/architect`。

## 1. 使用者是誰 / 情境 (Persona & Context)
- **誰**:CV / 缺陷判讀工程師(fab / 產線場景),每天要在**大量影像**中找問題、判定、回流模型資料。
- **影像特性**:wafer 表面 / 製程影像,常見 3000×3000 以上大尺寸、8-bit 與 16-bit grayscale、TIFF;
  缺陷型態包含 droplet、bubble、反光(reflection)、動態變化。
- **現況痛苦**:得在「檔案總管 / 圖片瀏覽器 / Excel / 截圖工具 / Labeling Tool」之間不停切換,
  工作流被工具切割得很碎。

## 2. 想達成的目標 (Jobs To Be Done)
一條主線:**找圖 → 放大判讀 → 標記可疑案例 → 比較前後影像 → 留下判定 → 回流模型資料集**。
- 我想要**在一堆圖裡很快找到重點**,而不是一張張翻。
- 我想要**像看 pixel 一樣放大**(滾輪定點縮放、看座標與 RGB/Gray 值、ROI 局部放大)。
- 我想要**標記「要回來看的圖」與「某個可疑區域」**,而且事後找得回來。
- 我想要**留下判定**(True Defect / False Alarm / Reflection …)和註解,而且**不破壞原圖**。
- 我想要**比較前後兩張影像**(左右 / 疊圖 / 閃爍 / 滑桿 / 差分),分清「固定反光 vs 真的在動」。
- 我想要**看到模型結果疊圖**(YOLO 框 / 信心 …),並把「模型結果 + 人工判定」一起保存下來。
- 我想要**把重要案例一鍵打包**成可分享、可回訓的證據。

## 3. 痛點 (Pain points)
- 圖太多、找不到重點;一張張翻很慢。
- 一般圖片瀏覽器**放大不夠專業**,看不準 pixel、看不到座標 / 灰階值。
- 看過之後**忘記哪張重要**,想回來看又找不到。
- **模型結果與人工判定沒有被保留**,討論時無據可查。
- 想比較前後影像、想看模型疊圖,都得另外開工具或截圖拼貼。

## 4. 使用者眼中的「成功」(可觀察判準)
- 開一個資料夾後,**3 秒內**看到縮圖牆,一眼能看出「這批圖有沒有長得不一樣」。
- 全程**鍵盤就能跑**:上一張 / 下一張 / 跳號 / 標記,不必一直摸滑鼠。
- 滾輪縮放**以游標為中心**;游標處**即時顯示 (x, y) 與 RGB/Gray 值**。
- 關掉工具明天再開,**自動回到上次看到的那一張**,且**進度顯示**像 `127 / 850`。
- 我對某張圖下的 Bookmark / Tag / Verdict / Comment / ROI,**之後一定找得回來**(可被搜尋與匯出)。
- 之後可以直接問出像「**YOLO 低信心 + 人工確認為真 defect**」這種案例清單。

## 5. 不在乎 / 可放棄的 (Non-goals,給 PO 砍的空間)
- **不是**完整 Labeling 平台,也**不是**一般看圖軟體。
- 第一版**不需要**:相似圖搜尋、DINO embedding cluster、SAM mask、多模型版本四格比較、
  自動 Missed-Detection Queue、FiftyOne/LabelMe/COCO 匯入匯出、HTML/PDF 報告。
  → 這些是**第二階段**,等第一版用順再加。
- 第一版**不需要**多人協作 / 帳號權限 / 雲端;單機、單人即可。

## 6. 第一版我自己最想先解決的(使用者視角的 MVP)
> 把「圖很多、找不到重點、放大麻煩、看過後忘了哪張重要、想回來看又找不到、
> 模型結果與人工判定沒被保留」這幾件每天最痛的事先解決掉,就非常實用。
1. 資料夾瀏覽 + 縮圖牆(Contact Sheet)
2. 超順暢 Zoom / Pan / Mini Map / Pixel 值
3. Bookmark、Tag、Comment、Review Status
4. ROI 框選與 Crop 匯出
5. 前 / 後影像比較與 Blink 模式
6. YOLO bbox / confidence overlay
7. 快捷鍵操作
8. 匯出 CSV + JSON Case List
